Implementação: Definir uma Métrica de Desempenho¶

É difícil medir a qualidade de um modelo dado sem quantificar seu desempenho durante o treinamento e teste. Isso é geralmente feito utilizando algum tipo de métrica de desempenho, através do cálculo de algum tipo de erro, qualidade de ajuste, ou qualquer outra medida útil. Para este projeto, você irá calcular o coeficiente de determinação, R2, para quantificar o desempenho do seu modelo. O coeficiente da determinação para um modelo é uma estatística útil em análise regressa, como se ele frequentemente descrevesse como "good" a capacidade do modelo de fazer estimativas.

Os valores para R2 têm um alcance de 0 a 1, que captura a porcentagem da correlação ao quadrado entre a estimativa e o valor atual da variável alvo. Um modelo R2 de valor 0 sempre falha ao estimar a variável alvo, enquanto que um modelo R2 de valor 1, estima perfeitamente a variável alvo. Qualquer valor entre 0 e 1 indica qual a porcentagem da variável alvo, ao utilizar esse modelo, ele pode ser explicado pelos atributos. Um modelo pode dar também um R2 negativo, que indica que o modelo não é melhor do que aquele que estima ingenuamente a média da variável alvo.

Questão 7 - Busca em Matriz¶

O que é a técnica de busca em matriz (grid search) e como ela pode ser aplicada para otimizar um algoritmo de aprendizagem?

Resposta:

Grid Search consiste em criar e testar um modelo para cada combinação possível dos parâmetros.

Voltando ao exemplo anterior imagine que você quer fazer um modelo de arvore de decisão e não sabe qual profundidade usar, você pode usar um Grid Search para testar as possibilidades dentro de um range, ou seja, treinar e testar repetidas vezes o modelo, cada vez utilizando um valor dentro do range como profundidade. Identificamos o a profundida ideal a partir da comparação do score obtido em cada tentativa feita, a profundidade que resultar no modelo com melhor score e comportamento em relação a erros de viés ou de variância é a escolhida.

Através do Grid Search podemos determinar qual a melhor combinação de parâmetros para um resultado mais preciso para um determinado modelo em uma determinada situação e dessa forma otimizar o modelo.

Questão 8 - Validação Cruzada¶

O que é a técnica de treinamento de validação-cruzada k-fold? Quais benefícios essa técnica proporciona para busca em matriz ao otimizar um modelo?
Dica: Assim como há um raciocínio por trás de utilizar um conjunto de teste, o que poderia dar errado ao utilizar busca em matriz sem um conjunto de validação cruzada?

Resposta:

O K-fold consiste em dividir um conjunto de dados em n partes de forma aleatória. Então se executa o treino e teste n vezes, cada uma das vezes se utiliza uma das n partes como teste e as demais como treino, cada uma das n vezes em que o modelo será treinado e testado e parte utilizada para teste é diferente. Dessa forma podemos validar se o modelo é eficaz independente da combinação entre dados de treino e dados teste selecionadas.

Além de validar o modelo pode ser treinado dessa forma, ou seja, realizar diversos treinos, cada um com um subconjunto dos dados e obter n estimativas. Para chegar a uma única estimativa final se tira a média das n estimativas obtidas.

A vantagem de utilizar esse modelo em conjunto com um Grid Search é que existe o risco de dentro do Grid Search dependendo da parte dos dados utilizada no treino os parâmetros podem ficar “viciados”, causando assim um caso de overgitting, ou seja, o modelo se sai muito bem com os dados de treino, mas não se sai bem em estimativas de outro dados. Dessa forma além de avaliar diversas combinações de parâmetros é possível testar o desempenho dessas combinações em combinações diferentes dos dados utilizados, isso traz mais segurança e assertividade na escolha do modelo.

# TODO: Importar 'make_scorer', 'DecisionTreeRegressor' e 'GridSearchCV'
from sklearn.tree import DecisionTreeRegressor 
from sklearn.model_selection import GridSearchCV 
from sklearn.metrics import make_scorer 

def fit_model(X, y):
    """ Desempenhar busca em matriz sobre o parâmetro the 'max_depth' para uma 
        árvore de decisão de regressão treinada nos dados de entrada [X, y]. """
    
    # Gerar conjuntos de validação-cruzada para o treinamento de dados
    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)

    # TODO: Gerar uma árvore de decisão de regressão de objeto
    regressor = DecisionTreeRegressor()

    # TODO: Gerar um dicionário para o parâmetro 'max_depth' com um alcance de 1 a 10
    params = {'max_depth':range(1,11)}

    # TODO: Transformar 'performance_metric' em uma função de pontuação utilizando 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Gerar o objeto de busca em matriz
    grid = GridSearchCV(regressor, params, scoring_fnc, cv = cv_sets)

    # Ajustar o objeto de busca em matriz com os dados para calcular o modelo ótimo
    grid = grid.fit(X, y)

    # Devolver o modelo ótimo depois de realizar o ajuste dos dados
    return grid.best_estimator_

Um modelo ótimo não é necessariamente um modelo robusto. Às vezes, um modelo é muito complexo ou muito simples para generalizar os novos dados. Resposta:

  • Quão relevante dados coletados em 1978 podem ser nos dias de hoje? Não. Pois tudo em relação aos dados análisados muda em relação ao tempo. As escolas mudam, as vizinhanças mudam e as casas ficam mais velhas. Este modelo tem uma vida útil de apenas alguns anos. Para ser valido hoje, na minha opinião, ele deveria ser treinado e atualizado constantemente durante os anos.
  • Os atributos presentes são suficientes para descrever um imóvel? Os atributos presentes são muito significativos, mas mesmo assim acredito que deveria haver mais alguns dados, como por exemplo: metódo construtivo, áera do imovel, tipo do imóvel, entre outros. Porém isso é apenas uma opinião, para ser ter certeza seria necessário adicionar outras informações e fazer alguns testes para definir qual dados impactam no resultado.
  • Esse modelo é robusto o suficiente para fazer estimativas consistentes? O modelo não precisa ser robusto para fazer estimativas consistentes. Dependendo da necissidade um modelo simples pode ser melhor do que um modelo mais robusto. No teste de sensibilidade este modelo mostrou uma variação dos preços de 69,044.61, essa variação fica na casa de 20% dos preços estimados. Dependendo da necessidade isso pode ser uma variação insignificante ou não. Neste caso acredito que esta variação é um pouco alta, acredito que se fosse adicionado mais dados essa variação poderia cair.
  • Dados coletados em uma cidade urbana como Boston podem ser aplicados para uma cidade rural? Não. Dados são validos para a região onde foram coletados, até porque a localização é uma das variaveis que são levadas em consideração pelo modelo.
